# Алтай на автодоме

Статичный интерактивный сайт-история о путешествии по Алтаю. Во время вертикальной прокрутки автодом движется по фактической траектории, камера следует за маршрутом, а остановки показывают фотографии и короткие тексты.

## Быстрый старт

Требуется актуальная LTS-версия Node.js.

```bash
npm install
npm run dev
```

Production-сборка:

```bash
npm run build
npm run preview
```

Готовые статичные файлы появятся в `dist/`. Каталог можно опубликовать на GitHub Pages, Cloudflare Pages, Netlify, Vercel Static или другом статичном хостинге.

## Команды

- `npm run dev` — локальная разработка.
- `npm run photos` — чтение EXIF и создание AVIF/WebP.
- `npm run validate-content` — проверка маршрута, сцен, файлов и `alt`.
- `npm run typecheck` — проверка TypeScript.
- `npm test` — автоматические проверки сценарного состояния.
- `npm run build` — полный фотопайплайн, валидация, проверка типов и сборка.
- `npm run preview` — локальный просмотр каталога `dist/`.

## Контент

```text
content/
  trip.json                 основные подписи путешествия
  scenes.json               порядок сцен, камера, фото и эффекты
  photo-overrides.json      ручные привязки и alt
  photo-report.json         автоматически создаваемый отчёт
assets/
  photos/                   исходные JPEG с EXIF
  generated/                исходный прозрачный ассет автодома
public/
  data/route.geojson        утверждённая упорядоченная траектория
  data/photo-manifest.json  автоматически создаваемый манифест
  photos/                   оптимизированные изображения
```

Все тексты, порядок, параметры камеры, композиции и эффекты меняются в JSON без правок основной логики.

## Добавление фотографий

1. Положите JPEG, PNG, WebP или поддерживаемый сборкой HEIC в `assets/photos/`.
2. Добавьте запись в `content/photo-overrides.json`. Идентификатор — имя файла в нижнем регистре без расширения.
3. Обязательно задайте содержательный `alt`. При необходимости задайте `sceneId`, `order`, `featured`, `hidden` или исправленные `coordinates`.
4. Запустите `npm run photos`.
5. Проверьте `content/photo-report.json`: туда попадают файлы без GPS, слишком далёкие от маршрута или не назначенные автоматически.

Порог автоматической привязки задаётся полем `photoAutoAssignThresholdKm` в `content/trip.json`. Публичные копии создаются в AVIF и WebP трёх размеров; EXIF в них не переносится.

## Добавление сцены

1. Добавьте объект в нужное место массива `content/scenes.json`.
2. Задайте уникальный человекочитаемый `id`; он станет ссылкой `?step=<id>`.
3. Укажите `routeProgress` от `0` до `1`, координаты, текст и параметры `camera`.
4. Выберите `layout`: `text-only`, `single`, `stack` или `sequence`.
5. Добавьте идентификаторы фотографий и синхронизируйте их `sceneId` в `photo-overrides.json`.
6. Запустите `npm run build`.

`routeProgress` должен возрастать вместе с порядком сцен. Возвратные участки уже повторены в `route.geojson` в фактическом порядке прохождения, поэтому обратная дорога имеет собственный диапазон прогресса.

## Карта

Карта работает на Yandex Maps JavaScript API 3.0. Получите ключ пакета «JavaScript API», создайте локальный `.env` и добавьте ключ:

```bash
cp .env.example .env
```

```text
VITE_YANDEX_MAPS_API_KEY=ваш_ключ
```

Для ключа обязательно настройте ограничение по HTTP Referer: разрешите локальный адрес разработки и домен опубликованного сайта. Ключ статичного сайта виден в браузере, поэтому не используйте его для других API.

Если ключ отсутствует, API или WebGL недоступны, сайт показывает локальную схему маршрута; тексты и фотографии продолжают работать.

## Публикация

1. Укажите базовый путь в `vite.config.ts`, если сайт публикуется не из корня домена.
2. Добавьте опубликованный домен в ограничение по HTTP Referer ключа Яндекс Карт.
3. После определения окончательного адреса добавьте абсолютный canonical URL и `sitemap.xml`.
4. Опубликуйте содержимое `dist/`.

Перед публикацией вручную проверьте мобильные Safari и Chrome, быстрый и обратный скролл, прямые ссылки, кнопки браузера, медленное соединение и `prefers-reduced-motion`.
